## 1. 기능 요건 (Functional Requirements)

> 우선순위 기준 — **P1 필수:** 없으면 앱이 동작하지 않음 / **P2 권장:** 없어도 최소 동작하나 핵심 UX 미충족

| ID | 이름 | 설명 | 우선순위 |
|----|------|------|---------|
| FR-01 | 게시글 목록 조회 및 검색 | 전체 게시글을 최신순으로 표시한다. 10개씩 클라이언트 페이지네이션을 적용하고, 24시간 이내 글에 NEW 배지를 표시한다. 키워드 입력 시 제목·내용을 동시에 검색하며, × 버튼으로 전체 목록으로 초기화할 수 있다. | **P1** |
| FR-02 | 게시글 상세 조회 | 게시글 제목·작성자·내용·작성일·조회수를 표시한다. 상세 페이지 진입 시 조회수를 +1 증가시킨다. 수정용 조회(`/edit`)는 조회수를 변경하지 않는다. | **P1** |
| FR-03 | 게시글 작성 | 작성자(1–50자)·제목(1–200자)·내용을 입력해 게시글을 등록한다. 필수 항목이 비어 있으면 클라이언트와 서버 양쪽에서 오류 메시지를 표시하고 등록을 거부한다. 등록 성공 시 목록 페이지로 이동한다. | **P1** |
| FR-04 | 게시글 수정 | 기존 게시글의 제목·내용을 불러와 수정하고 저장할 수 있다. 저장 성공 시 상세 페이지로 이동하며 조회수는 변경되지 않는다. 유효성 규칙은 FR-03과 동일하게 적용한다. | **P2** |
| FR-05 | 게시글 삭제 | 삭제 버튼 클릭 시 확인 모달을 표시한다. 모달에서 확인을 누르면 즉시 영구 삭제하고 목록 페이지로 이동한다. 취소를 누르면 상세 페이지를 유지한다. | **P2** |

## 2. 비기능 요건 (Non-Functional Requirements)

### 2-1. 성능 (Performance)

| ID | 항목 | 측정 기준 |
|----|------|---------|
| NFR-P1 | API 응답 시간 | `GET /api/posts` 응답 시간 **≤ 500 ms** (p95 기준, 게시글 500건 이하 환경) |
| NFR-P2 | 페이지 최초 콘텐츠 표시 | 목록 페이지 FCP(First Contentful Paint) **≤ 2초** (유선 인터넷, 크롬 기준) |

### 2-2. 보안 (Security)

| ID | 항목 | 측정 기준 |
|----|------|---------|
| NFR-S1 | XSS 방지 | 모든 사용자 입력을 HTML 이스케이프 처리. `<script>` 태그를 포함한 게시글을 등록했을 때 브라우저에서 스크립트가 실행되는 건수 **= 0** |
| NFR-S2 | 입력 유효성 오류 응답 | 제목 0자 또는 201자 이상, 작성자 0자 또는 51자 이상 입력 시 서버가 **HTTP 400** 반환; 존재하지 않는 ID 조회·수정·삭제 시 **HTTP 404** 반환 |

### 2-3. 가용성 (Availability)

| ID | 항목 | 측정 기준 |
|----|------|---------|
| NFR-A1 | 월간 업타임 | 서비스(앱 + Supabase DB) 월간 가용률 **≥ 99.0%** (다운타임 ≤ 7.3 h/월) |
| NFR-A2 | 서버 오류율 | 월간 전체 API 요청 대비 5xx 응답 비율 **≤ 1%** |

### 2-4. 접근성 (Accessibility)

| ID | 항목 | 측정 기준 |
|----|------|---------|
| NFR-AC1 | 키보드 완전 조작 | 마우스 없이 Tab · Enter · Esc 키만으로 목록 탐색 → 상세 조회 → 글쓰기 → 제출 전 과정을 오류 없이 완료할 수 있어야 한다 (수동 테스트 통과 기준) |
| NFR-AC2 | 색상 명도 대비 | 본문 텍스트와 배경색의 명도 대비 비율 **≥ 4.5 : 1** (WCAG 2.1 Level AA · Chrome DevTools Accessibility 패널 기준) |
