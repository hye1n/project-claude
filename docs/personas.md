## 2. MVP 헌장

> **원칙:** 단일 테이블 · 기능 5개 이하 · 1일(8 h) 안에 코드로 옮길 수 있는 분량

### 2-1. 테이블

```
posts
─────────────────────────────────
id          BIGSERIAL PK
title       VARCHAR(200) NOT NULL
content     TEXT
author      VARCHAR(50)  NOT NULL
view_count  INTEGER DEFAULT 0
created_at  TIMESTAMPTZ  (자동)
updated_at  TIMESTAMPTZ  (자동)
```

### 2-2. MVP 기능 5개

| # | 기능 | 엔드포인트 | 대상 페르소나 |
|---|------|-----------|------------|
| F-1 | 게시글 **목록 조회** (최신순, 검색 포함) | `GET /api/posts[?keyword=]` | A · B · C |
| F-2 | 게시글 **상세 조회** (조회수 자동 증가) | `GET /api/posts/{id}` | A · B · C |
| F-3 | 게시글 **작성** | `POST /api/posts` | A · B · C |
| F-4 | 게시글 **수정** (조회수 증가 없음) | `PUT /api/posts/{id}` | A · B |
| F-5 | 게시글 **삭제** (확인 모달) | `DELETE /api/posts/{id}` | B |

### 2-3. MVP 범위 밖 (v2 이후)

- 로그인 / 회원 인증 (현재 작성자명은 자유 입력)
- 댓글 테이블 추가
- 서버사이드 페이지네이션
- 파일 첨부 / 이미지 업로드
- 관리자 권한 분리

### 2-4. 완료 기준 (Definition of Done)

- [ ] 5개 엔드포인트가 모두 HTTP 200/201/204를 반환한다.
- [ ] 빈 제목·작성자 입력 시 클라이언트·서버 양쪽에서 오류 메시지를 표시한다.
- [ ] `<script>` 태그를 포함한 게시글이 화면에 그대로 출력되지 않는다(XSS 방지).
- [ ] 로컬(`localhost:8080`) 및 Supabase 연결 상태에서 정상 동작한다.
